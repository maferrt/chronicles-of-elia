# MVP Content Plan

## Overview

This document defines the initial educational content plan for the Chronicles of Elia MVP.

The goal is not to create a complete English course yet.

The goal is to create enough structured content to prove the main learning experience:

```text
User profile
↓
Personalized path
↓
Contextualized mission
↓
Lesson
↓
Practice
↓
Progress
```

## MVP Content Strategy

The MVP will focus on a small but meaningful content seed.

Instead of creating many incomplete lessons, the MVP will create a small number of well-structured missions that demonstrate the pedagogical model.

The first content seed will focus mainly on:

```text
Level: A2
Primary Path: DEV
Main Goals:
- Technical Interviews
- Remote Work
- Professional Communication
```

Secondary paths will be included only as basic proof of personalization:

```text
A2 + CHEF
A2 + ARTIST
```

## Why A2 First?

A2 is a strong starting point for the MVP because it allows the learner to use simple but meaningful professional English.

A1 can be too limited for professional communication.

B1 is more useful for professional independence, but requires more complex content.

A2 allows missions such as:

- Introducing yourself professionally
- Describing your work routine
- Asking for help
- Talking about a simple project
- Explaining a basic problem

Example A2 structures:

```text
I am a junior developer.
I work with Java.
I usually write documentation.
I have to fix a bug.
Can you help me?
I worked on a small project.
```

## CEFR-Inspired Level Rule

Each level is treated as a long-term learning path.

For internal progress tracking:

```text
1 CEFR-inspired level = 500 study hours
500 hours = 30,000 study minutes
```

The MVP will not provide complete mastery of a level.

The MVP will only demonstrate the beginning of a personalized level journey.

Example:

```text
A2 target: 30,000 minutes
Mission completed: 30 minutes
A2 progress: 30 / 30,000 minutes
```

## Mission 1: Professional Introduction

### Basic Information

```text
Mission Title:
Professional Introduction

Level:
A2

Path:
DEV

Learning Goal:
Technical Interviews

Estimated Time:
30 minutes

XP Reward:
50 XP
```

### Communicative Objective

The learner can introduce herself as a junior developer using simple and clear English sentences.

### Function Focus

- Giving personal information
- Describing role
- Describing basic work interests

### Grammar Focus

- Verb to be
- Present simple
- Basic sentence structure
- Simple connectors: and, but

### Vocabulary Focus

| Word/Phrase | Meaning |
|---|---|
| role | The position or function someone has in a team or project |
| junior developer | A developer at an early professional level |
| backend | The server-side part of an application |
| frontend | The user-facing part of an application |
| full stack | Frontend and backend development |
| project | A planned piece of work |
| team | A group of people working together |
| interested in | Used to talk about something you want to learn or do |

### Lesson Content

When you introduce yourself professionally, you can mention:

1. Your name
2. Your role
3. The technologies or tools you use
4. What you are learning
5. Your professional goal

### Examples

```text
My name is Mafer.
I am a junior full stack developer.
I work with Java, Spring Boot and TypeScript.
I am interested in backend development.
I want to improve my English for technical interviews.
```

### Elia's Tip

```text
Start simple. A clear introduction is better than a long and confusing one.
```

### Practice Activities

#### Activity 1: Multiple Choice

Question:

```text
Choose the correct sentence.
```

Options:

```text
A. I are a junior developer.
B. I am a junior developer.
C. I be a junior developer.
D. I is a junior developer.
```

Correct answer:

```text
B
```

Feedback:

```text
Correct! We use "am" with "I".
```

#### Activity 2: Fill in the Blank

Question:

```text
Complete the sentence.
```

Sentence:

```text
I ____ with Java and Spring Boot.
```

Options:

```text
A. work
B. works
C. working
D. worked
```

Correct answer:

```text
A
```

Feedback:

```text
Correct! With "I", we use the base form: "I work".
```

#### Activity 3: Sentence Ordering

Instruction:

```text
Put the words in the correct order.
```

Words:

```text
am / I / full stack / junior / developer / a
```

Correct sentence:

```text
I am a junior full stack developer.
```

#### Activity 4: Final Task

Instruction:

```text
Write a short professional introduction.
```

Expected answer example:

```text
My name is Mafer. I am a junior full stack developer. I work with Java and TypeScript. I am interested in backend development.
```

### Success Criteria

The answer should include:

- Name
- Role
- At least one technology or tool
- One professional interest or goal

## Content-to-Database Translation

The educational content will later be represented in the database using entities such as:

```text
missions
lessons
vocabulary_items
exercises
exercise_options
user_mission_progress
user_exercise_answers
```

Each mission should store:

```text
title
description
communicative_objective
cefr_level_code
profession_code
learning_goal_code
function_focus
grammar_focus
vocabulary_focus
estimated_minutes
xp_reward
order_index
```

Each lesson should store:

```text
mission_id
title
content
elia_tip
order_index
```

Each exercise should store:

```text
mission_id
lesson_id
type
question
correct_answer
feedback
order_index
```

Each exercise option should store:

```text
exercise_id
option_text
is_correct
order_index
```

## MVP Completion Criteria

The content MVP is complete when the backend can:

- Store initial missions
- Store lessons for each mission
- Store vocabulary items
- Store basic exercises
- Return missions by user profile
- Return mission detail
- Return lesson content
- Submit exercise answers
- Save basic progress
- Show completed missions and XP

## Content Philosophy

The app should always prioritize:

```text
clear language
real context
short explanations
useful feedback
professional relevance
long-term progress
```

Fantasy elements should make the experience memorable, but they should not replace clear teaching.
